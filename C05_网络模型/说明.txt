网络模型 = 多层神经网络结构 + 训练得到的参数（权重+偏置）
结构是你手动设计的（比如几层、每层多少神经元、用什么激活函数、是否用残差连接等）。
参数(卷积核里面的数字等)是训练过程中自动学习出来的，用来拟合你的数据。
不同任务（图像、文本、时间序列、图结构）确实需要不同的结构，比如：
图像 → CNN / Transformer
文本 → RNN / Transformer
图数据 → GNN
时序 → LSTM / TCN / Transformer

那么如何为“自己的情景”找到或设计一个合适的模型呢？

| 任务类型   | 常见输入     | 推荐结构                                      |
| ------ | -------- | -------------------------------------------       |
| 图像分类   | 图片       | CNN（ResNet、EfficientNet）、Vision Transformer |
| 文本分类   | 句子/段落    | BERT、RoBERTa、LSTM                           |
| 时间序列预测 | 传感器数据、股价 | LSTM、TCN、Transformer                     |
| 图结构数据  | 社交网络、分子  | GCN、GAT、GraphSAGE                         |
| 多模态    | 图文混合     | CLIP、BLIP、自定义Transformer                   |

注意最好要从“已有模型”开始，而不是从零设计
不要从零设计结构！ 先找一个和你任务最接近的开源模型，然后：
    微调（fine-tune）：用你的数据继续训练。
    剪枝/蒸馏：如果模型太大，做轻量化。
    结构调整：小幅修改（比如加一层、改激活函数、换损失函数）。

只有在你确定现有模型都不合适时，才考虑：
    NAS（神经架构搜索）：用工具比如 Optuna、NNI、KerasTuner 自动搜索结构。
    手工设计：基于你对任务的理解，尝试：
        加深/加宽网络
        引入残差连接、注意力机制
        多尺度特征融合
        自定义损失函数（比如加入领域知识）
